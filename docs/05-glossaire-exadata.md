# Glossaire Exadata

| Terme | Définition |
|---|---|
| **ASM** | Automatic Storage Management, couche Oracle de gestion des disques et diskgroups utilisée par Oracle Database. |
| **CellCLI** | Interface de commande d’administration et de lecture pour les cellules Exadata Storage Server. |
| **Cell Disk** | Objet de stockage physique ou logique côté cellule, à partir duquel sont créés des grid disks. |
| **Grid Disk** | Objet exposé par une cellule Exadata et consommé par ASM pour constituer des diskgroups. |
| **Smart Scan** | Mécanisme d’offload qui permet à certaines opérations SQL d’être traitées côté cellule storage. |
| **IORM** | I/O Resource Management, mécanisme de gouvernance des ressources I/O entre workloads consolidés. |
| **OEDA** | Oracle Exadata Deployment Assistant, outil de préparation et génération de configuration selon contexte/version. |
| **OECA** | Oracle Exadata Configuration Assistant, outil ou référence de configuration selon génération et documentation. |
| **AHF** | Autonomous Health Framework, famille d’outils de diagnostic et santé fournie par Oracle. |
| **Exachk** | Outil de vérification de santé et bonnes pratiques pour environnements Oracle Engineered Systems. |
| **ASR** | Auto Service Request, mécanisme de création automatique de demande de service lorsqu’il est configuré et applicable. |
| **DBFS** | Database File System, système de fichiers basé sur Oracle Database, cité pour certains scénarios de chargement massif. |
| **RoCE** | RDMA over Converged Ethernet, technologie réseau utilisée par certaines générations Exadata. |
| **InfiniBand** | Technologie réseau interne utilisée par des générations Exadata antérieures ou selon modèle. |
| **Cloud@Customer** | Modèle Oracle permettant d’exécuter des services cloud Oracle dans le datacenter client. |
