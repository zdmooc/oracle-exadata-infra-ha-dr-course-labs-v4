# Contribuer au cours

Merci de contribuer à ce support de cours Oracle Exadata. Les contributions doivent améliorer la clarté pédagogique, la précision technique, la reproductibilité des labs et la sécurité des opérations.

## Règles éditoriales

Les contributions doivent utiliser un style professionnel, expliciter les prérequis, éviter les commandes destructives non encadrées et citer les sources officielles lorsque des fonctionnalités Oracle sont décrites. Les labs doivent toujours indiquer l’objectif, la durée, les prérequis, les étapes, les commandes indicatives, les critères de validation, les erreurs fréquentes et les actions de nettoyage.

## Processus proposé

| Étape | Attendu |
|---|---|
| Issue | Décrire le problème ou l’amélioration pédagogique. |
| Branche | Utiliser un nom explicite comme `docs/add-iorm-lab` ou `labs/dataguard-runbook`. |
| Pull request | Expliquer l’impact, les fichiers modifiés et les validations effectuées. |
| Relecture | Vérifier exactitude technique, sécurité et cohérence du parcours. |
